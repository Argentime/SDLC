## 1. Введение

Данный документ описывает план тестирования для веб-приложения «Freelance Marketplace». Документ предназначен для разработчиков и тестировщиков, участвующих в создании и проверке системы. Цель тестирования — проверка соответствия реального поведения программы и ее ожидаемого поведения, которое описано в документе с требованиями (SRS).

<a name="items"/>

## 2. Объект тестирования

В качестве объектов тестирования можно выделить следующие функциональные требования:

*   Регистрация нового пользователя;
*   Доступ к приложению (авторизованный и гостевой);
*   Настройка профиля пользователя;
*   Загрузка и просмотр списка фрилансеров;
*   Просмотр подробной информации о фрилансере;
*   Выход из системы.

<a name="quality"/>

## 3. Атрибуты качества

1.  **Функциональная пригодность:**
    *   **Функциональная полнота:** приложение должно выполнять все заявленные в SRS функции.
    *   **Функциональная корректность:** приложение должно выполнять все заявленные функции корректно.
2.  **Удобство использования:**
    *   Все ключевые функции доступны и работают корректно.
    *   Элементы интерфейса имеют понятные названия, отражающие их функции.
    *   Интерфейс минималистичен, без избыточных деталей.
3.  **Безопасность:**
    *   Доступ к данным для редактирования ограничен текущим авторизованным пользователем.

<a name="risk"/>

## 4. Риски

К основным рискам можно отнести:
*   Нестабильная работа или отсутствие доступа к удаленной базе данных PostgreSQL.
*   Некорректная обработка пользовательских данных, что может привести к уязвимостям безопасности.
*   Снижение производительности при большом количестве записей о фрилансерах в базе данных.

<a name="features"/>

## 5. Аспекты тестирования

В ходе тестирования планируется проверить реализацию основных функций приложения. Аспекты, подвергаемые тестированию: 
*   Регистрация нового пользователя;
*   Авторизация пользователя;
*   Гостевой доступ;
*   Редактирование профиля;
*   Просмотр списка фрилансеров;
*   Выход из системы.

### Регистрация нового пользователя
Этот вариант использования необходимо протестировать на:
*   Успешное создание нового аккаунта с корректными данными;
*   Отображение ошибки при попытке регистрации с уже существующим email.

### Авторизация пользователя
Этот вариант использования необходимо протестировать на:
*   Успешный вход в систему с корректными данными;
*   Отображение ошибки при вводе неверного логина или пароля.

### Гостевой доступ
Этот вариант использования необходимо протестировать на:
*   Возможность просмотра списка фрилансеров без авторизации.

### Редактирование профиля
Этот вариант использования необходимо протестировать на:
*   Добавление/удаление навыка;
*   Редактирование личных данных.

### Просмотр списка фрилансеров
Этот вариант использования необходимо протестировать на:
*   Корректную загрузку и отображение списка фрилансеров;
*   Переход к странице с подробной информацией о фрилансере.

### Выход из системы
Этот вариант использования необходимо протестировать на:
*   Корректный выход из аккаунта и возврат на главную страницу.

### Нефункциональные требования
*   Элементы интерфейса имеют понятные названия и соответствуют [стандартам Material UI](https://material-ui.com).
*   Система быстро реагирует на действия пользователя.
